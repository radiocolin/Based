Smarter networking for poor connectivity

This build reworks how the app polls for live game updates so it behaves better on slow, degraded, or metered connections.

What changed
- The app now uses Link Quality (new in iOS 26) to detect when your connection is weak. On a poor link, it automatically backs off polling frequency so requests aren't wasted on a struggling connection.
- Polling intervals are now measured from when the last response finishes, not on a fixed clock. If a response takes 5 seconds on a slow network, the next request won't fire until after a full interval has passed — instead of nearly immediately.
- The app tracks how long recent requests are taking. If responses are consistently slow, it spaces out requests further to avoid piling on.
- Low Data Mode still doubles all intervals. These adjustments layer together but are capped so the app stays responsive even in the worst conditions.

What to test
- Use the app on Wi-Fi, then switch to cellular mid-game and confirm updates keep flowing without long gaps or freezes.
- If you can test on a weak connection (e.g. one bar of signal, crowded stadium Wi-Fi), check that the app stays usable and doesn't hang or spam failed requests.
- Turn on Low Data Mode (Settings > Cellular > Cellular Data Options > Data Mode > Low Data Mode) during a live game. Updates should still arrive, just less frequently.
- Background the app for a minute or two during a live game, then return. The score and timeline should refresh promptly.
- Watch a game from start to finish. Confirm inning transitions look clean — no missing final outs or duplicate at-bats when innings change.

General
- Report any case where the app feels unresponsive, shows stale data for too long, or drains noticeably more battery than expected.

Live state + stale data fixes

This build adds smarter behavior for non-active games and better protection against stale UI data after switching dates/games.

What changed
- Final game detection now uses both linescore and `/feed/live` status (`final`, `game over`, `completed early`, plus `f/o` status codes), and polling stops once final is confirmed.
- Pregame polling now adapts to scheduled first pitch: far-away starts are polled much less often, with a periodic pregame lineup probe so posted lineups still appear before warmup.
- Added polling-session guards so canceled/old polling tasks cannot apply snapshots after switching games/dates.
- Schedule date changes now clear old game cards immediately while loading the new date, reducing stale content carryover.

What to test
- Open a pregame matchup several hours before first pitch and verify the app stays quiet but still eventually picks up lineup availability.
- Switch dates repeatedly in the schedule and confirm old-date cards/content do not persist while new data loads.
- Enter one game, back out, quickly open another game, and confirm no scorecard/timeline/header data leaks from the prior game.
